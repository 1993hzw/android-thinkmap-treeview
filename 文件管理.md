一、目录结构
Linux文件的目录结构类似一棵倒置的树，以一个名为根(/)的目录开始向下延申
分析目录树：
	/bin:binary的缩写，主要存放普通用户的一些命令
	/sbin:s指的是super user的意思，这个目录下存放的主要内容是系统管理使用的命令程序
	/boot：主要存放的是内核及系统启动所需要的文件
	/dev：device的缩写 这个目录下放的是设备文件
	/home:普通用户的家目录，用户的个人数据存放在该目录中
	/lib lib64:主要存放系统最基本的函数文件。如核心模块、驱动等。所有应用都会用到此库
	/lost+found:存放fsck用的孤立文件。平时为空，当系统不正常关机，这里可能会出现文件片段，多在ext文件系统中存在
	
	/media：类似于mnt主要用途为挂载优盘等媒体介质
	/mnt：临时挂载别的文件系统的地方，比如挂载windows的NFFS/FAT格式文件系统
	/opt：这个目录主要用来安装可选用的应用程序，是第三方工具使用的安装目录
	/proc：这是一个虚拟文件系统，包含系统核心信息，里面包含许多随系统启动而启动的进程以及内存 CPU 硬盘等硬件的状态信息。这些信息随系统关闭而消失，随系统启动而产生
	
	/src：存放服务启动后需要的服务文件
	/sys：此目录问系统一个特殊的目录。在2.6内核之后变化大。sysfs（procfs devfs devpts）
	/tmp: 存放临时文件，这个目录需要经常清理
	/usr: UNIX Software Resource, 存放与用户相关的文件与目录，此目录中的内容一般在安装系统的过程中产生 
	/var：存放系统执行过程中经常变化的文件，比如邮件 日志 新闻等信息
	
	绝对路径与相对路径：
		绝对路径：由根目录‘/’开始写起的文件名或目录
		相对路径：相对于当前文件名或目录的写法
		.=./   当前目录
		..=../ 父目录
		
二、查看目录内容 cd pwd ls nautilus
	1 cd(change directory)
		cd /etc/sysconfig 进入指定目录
		cd .. 进入父目录
		cd . 进入当前目录
		cd 进入用户家目录
		cd ~ 进入用户家目录
		cd - 返回进入当前目录之前所在的目录
		cd !$ 把上一个指令的参数作为cd的参数使用
	2 pwd：print working directory 查看工作目录完整路径
		pwd -P 目录如果有链接显示实际路径而非链接路径
	3 ls:list directory contents 列出目录内容
		ls：显示当前目录下的内容（不包含隐藏文件和目录）
		ls -a 表示显示当前目录下的所有内容（不包含隐藏文件和目录）
			以‘.’开始的文件
		ls -A 表示显示当前文件夹的隐藏文件
		ls -l 以长格式显示当前文件的内容
		第一列
		
			蓝色-目录
			绿色-可执行
			灰色黑色-普通文件
			红色-压缩文件
			黄色-设备文件
			      字符设备文件c
				  块设备 b
				  两者区别：本质区别是否可以被随机访问
			xxx xxx xxx :
				  前三位表示所有者的权限
				  中间三位表示所有组的权限
				  最后三位表示其他人的权限
		第二列
			文件或目录的链接数
		
		第三列
			表示文件的所有者
			
		第四列
			文件的所有组
		第五列
			文件的大小，针对文件目录来说，仅表示目录所占的磁盘大小
		第六列
			文件的修改时间
		第七列
			文件或目录名称
		
		常用命令示例：
		ls /etc 显示指定目录下的文件和目录
		ls -lh=ls -l -h 以易读方式显示文件目录和大小		
		ls -lh -si 以1000为单位显示文件目录和大小
		ls -d */ 只显示当前目录下的所有目录信息
		ls -lg 不显示所有者信息
		ls -lG 不显示所属组信息
		ls -ln 以文件或目录的UID及GID显示长格式信息
		ls -i 显示n个文件或目录的索引号
		ls -p 增加/标记目录
		ls -F 增加文件类型显示符
		ls -R 表示递归列出目录中的文件或目录
		ls -r 按字母顺序反转排序
		ls ~ 显示用户家目录下内容
		
		时间概念
		access 访问时间：more cat head tail
		modify 修改时间 vi vim ls -lt
		change 属性变更时间 chmod chown ls -ltc
		
		
三、查看文件内容 more less cat tac nl head tail wc
	1 more 一页一页地显示文件内容
	语法：more [选项] [文件名]
	测试：
		命令：空格：向下翻页
		b:向上翻页
		Enter：向下一行行显示
		q：退出
		=：输出当前页中最后一行在文件的行数
		v：调用vi编辑器
		
	选项：
		-s:多个空白行显示为一个空空白行
		+NUM：表示从第几行开始显示
		-NUM：表示一页显示几行
		+/string：在每个文件显示之前先搜索指定的string字符串然后从该字符串的前后两行开始显示
		ls -l /etc/ | more :翻屏显示前一个命令输出的结果

	2 less 一页一页地显示文件内容让我们查看文件内容更加有弹性
	  语法：less [选项] [文件名]
	  测试：
		命令：
			空格：向下翻页
			b:向上翻页
			Enter：向下一行行显示
			q：退出
			=：输出当前页中最后一行在文件的行数/总行数；显示字节数/总字节数
			v：调用vi编辑器
			/字符串：向下搜索字符串功能，n键表示向下继续搜索，N键表示向上搜索
			?字符串：向上搜索字符串功能，n键表示向上继续搜索，N键表示向下搜索
			j：向后移动一行
			k:向前移动一行
			G：移动到文件最后一行
			g：移动带文件第一行
		
		选项：
			-s:多个空白行显示为一个空空白行
			+NUM：表示从第几行开始显示
			+/string：在每个文件显示之前先搜索指定的string字符串然后从该字符串的前后两行开始显示
			ls -l /etc/ | less :翻屏显示前一个命令输出的结果
			-i 搜索时忽略大小写
			-N 显示每行行号
			
		补充：
			ctrl+F 后移一屏
			ctrl+B 前移一屏
			ctrl+D 下移一屏
			ctrl+U 上移一屏
			
			
	3 cat 查看文件内容 同时可以打开多个文件
	  语法 cat [选项] filel file2 ---
	  三大功能：
		1）一次性显示整个文件 cat filename
		2）从键盘创建一个文件 cat > filename
		3）将几个文件合并为一个文件显示，并作重定向 cat file1 file2 > filename
	  选项：
		-b: 对文件行数进行编号
		-n：对文件所有行进行编号
		-s：对连续两行空白行的时代替为一行
		特殊应用：
		通过here doc来生成文件
		cat > log.file <<end
		>hello
		>good
		>byte
		>end
	4 tac 与cat相反将文件从最后一行开始倒过来显示输出到屏幕上
	5 nl 给文件名显示出行数 空白行不编号类似cat -b
	6  head 显示文件开头内容至标准输出（默认显示开头10行）
	语法：head [选项] [文件名]
		测试：
		head -n NUM filename 显示文件的前NUM行
		head -n -NUM filename 显示文件出来最后NUM行的内容
		head -c NUM filename 显示文件前NUM个字节
		head -c -NUM filename 显示文件除了最后NUM个字节外的所有内容
	7 tail 显示文件末尾内容至标准输出（默认最后10行）
	语法：tail [选项] [文件名]
	测试：
		tail filename 从末尾显示10行
		tail -n NUM filename 显示文件的末尾NUM行
		tail -n +NUM filename 从第NUM行开始显示内容
		tail -f filename 动态查询文件中显示的最新内容
		tail -n 5 -f /var/log/messages 
	8 wc --work characters 对文件内容进行统计的命令，行数单词数 字节数
	字节：计算机中基本计算单位，一般用于存储及数据传输
	字符：数字 字母 特殊符号
	1bite=1或0 二进制数据
	1byte=8bite
	1字母=1byte
	1汉字=2byte
	在计算机中有两种编码方式
		ANSI 标点符号 数字大小写字母都占一个字节，汉字占两个字节
		UNICODE 所有字符都占两个字节
	wc  显示的内容为 行数 单词数 字节数 文件名
	wc -l 统计文件行数
	wc -w 统计单词数
	wc -c 统计字节数

六 文件与目录的创建 复制 删除 转移及重命名， touch mkdir rmdir mv rm cp
	1 touch
		功能：
			1)修改文件或目录的时间，包括存取时间（access）和修改时间（modifiecation）
			2)如果当前目录中没有相关文件，则创建一个空文件
		语法：
			1)touch file
			2)touch [option] file
		选项：
			touch file:
				a.当所有操作的目录下没有file文件时，创建一个空文件
				b.当前操作的目录下有file文件时，则file文件的access/mofify/change属性都会发生变化
				-a 更改文件的存取时间access属性 配合-t使用
				-t 使用指定的日期给文件定义时间属性
				-m 更新文件的修改时间 配合-t使用
		
		小结：对文件的时间属性进行操作时，无论时操作access modify属性，其change时间都会发生变化，这种变化一般情况是跟系统当前时间一致的
		
		-r 指把文件或目录的时间设置成参考文件或目录的日期时间相同，需要注意的是-r后面跟的文件为参考文件，需要改变时间值的文件放在后面
		
	2 mkdir 创建目录
		语法：mkdir [option] 目录
		测试：
			mkdir dirname 在指定的路径下创建一个空目录dirname
			mkdir -p dirname 一次性可以创建多个目录
			mkdir -m dirname 对新建立的目录设置存取权限 chmod
	3 rmdir 删除空目录
		-p 递归删除多个空目录
	
	4 mv 更改文件目录或者重命名
	   mv a b 把a文件改名为b
	   mv b /root/bb 移动文件到指定目录下
	   
	   选项：
		-i 交互式操作 会有操作提醒
		-f 禁止交互操作 不给任何提示
		-b 覆盖会先备份
		-S 与-b选项一并使用，指定备份文件所要附加的字符串
		-u 在移动或更改文件名时，若目标文件已经存在，其文件日期比源文件新，则不覆盖
		
	5 rm 删除文件或目录
		语法：rm [option] 文件或目录
		测试：
		rm 55 删除55这个文件
		rm -rf 非常危险的操作，删除指定目录及子目录的所有内容
		选项：
			-i 交互式操作
			-f 强制删除文件或目录
			-r 递归处理
			
	6 cp 该命令可以复制一个或多个文件后者目录到另外一个目录中
	 语法：cp [option] [source file or dir] [destination file or dir]
	 测试：
		 cp -r /boot /root/ 把源文件/boot 复制到/root目录中
		 cp -i /44 /root/44 如果重复会有安全提示
	 选项：
		-r或-R如果给出的源文件是一个目录文件，此cp将递归复制子文件夹到目标文件中
		-i 安全提示
		-f 强制覆盖
		-u 如果目标文件较新则不覆盖
		-a  该 选项通常在复制目录时使用 ，它保留链接 文件属性并递归地复制目录，作用等同于-dpR
		
七 文件搜索 find locate which whereis type
	1 find 在目录中搜索文件, 并可以执行指定的操作，可以做到按文件名 文件类型 时间戳 权限等查找
	语法：find [路径] [查找条件] [操作]
	测试：
		1）根据文件名查找
		 find /etc /tmp /root -name passwd #在/etc /tmp /root目录下按名字查找passwd文件
		 find /etc -iname passwd #在/etc目录下按名字查找passwd文件,并忽略大小写
		 find /etc -iname “*sswd” #在/etc目录下按名字查找后四个字母为’sswd‘的文件
		 find /etc -iname “?sswd” #文件名为5个字母，并后四个字母为’sswd‘的文件
		4）-a (and) -o(or) -not在查找命令中使用
		 find /tmp -name “*.txt” -a -user root #文件扩展名为txt并且用户为root		 
		 find /tmp -name “*.txt” -o -user root #文件扩展名为txt或者用户为root
		 find /tmp -name “*.txt” -not -user root #文件扩展名为txt并且用户不为root
		5）根据时间戳查找
			-atime 对文件的访问时间戳 以天为单位 cat more less tac
			-mtime 对文件的修改时间 以天为单位 vi vim
			-ctime 对文件的属性修改时间 天为单位
			-amin 对文件的访问时间戳 分钟为单位
			-mmin 对文件的修改时间 分钟为单位
			-cmin 对文件的属性修改时间 分钟为单位
			
			find /home -atime -1 #在home中查找1天内访问过的文章或目录
			find /home -atime +1 #在home中查找1天内访问过的文章或目录
			find /home -mtime -2 #在home中查找最近来两天修改过的文章或目录
			find /home -mtime -2 #在home中查找两天以前修改过的文章或目录
			find /home -amin -2 #在home目录中查找两分钟内被访问过的文件或目录
			find /home -mmin -2 #在home目录中查找两分钟内被修改过的文件或目录
		6）根据文件类型查找文件
			普通文件 f 	
			目录文件 d 
			链接文件 l 
			块设备文件 b
			字符设备文件 c
			管道文件 p
			socket文件 s
			find /tmp -name *.txt -type f #在/tmp目录中查找扩展名为txt的文件
			find /tmp -name *.txt -type d #在/tmp目录中查找扩展名为txt的文件夹
		7）根据文件权限查找文件
			find /tmp -perm 755 在/tmp下查找权限为755的文件或目录
		8）根据文件大小查找文件
			 find  /tmp -size 2M #在tmp目录下查找大小为2M的文件或目录
			 find  /tmp -size +2M #在tmp目录下查找大于为2M的文件或目录
			 find  /tmp -size -2M #在tmp目录下查找小于为2M的文件或目录
			 find  /tmp -size 2K #在tmp目录下查找大小为2K的文件或目录
			 find  /tmp -size +2K #在tmp目录下查找大于为2K的文件或目录
			 find  /tmp -size -2K #在tmp目录下查找小于为2K的文件或目录
		 9)nouser 和 nogroup
			 find /home -nouser #查找不属于本地用户的文件或目录
			 find /home -nogroup #查找不属于本地组的文件或目录
			 find /home -nouser -a 	-nogroup		 #查找不属于本地组并不属于本地用户的文件或目录
		10）查找命令执行之后执行相关的动作
			-print #默认动作
			-ls #查找后的结果用ls显示
			-ok [命令] 查找执行命令时，询问用户使要进行
			-exec [命令] #查找后执行命令时不询问，直接执行
	2 locate 
		优点：查找速度快（/var/lib/mlocate/mlocation.db）
			  查找属于模糊查找，内容比较多
		确定：查找内容不精确，对最新产生的文件会目录查找功能欠缺
	3 which [命令]
		显示一个命令的完整路径与别名。在PATH环境变量指定的路径中，搜索某个命令的路径，并返回一个结果
		
	4 whereis [选项] [文件名]
		搜索一个命令的完整路径以及文件

八、文件操作命令 grep sed tr
	1 grep(grobal regular expression print)
		文本搜索工具，可对文件内容精确搜索
		语法：grep [option] [model] [file; file2;--]
			 a.普通搜索
				选项：
					grep good /tmp/* 搜索指定目录下所有文件中带有good的行但不含子目录
					-r 明确要求搜索子目录
					-c 输出搜索内容匹配行数
					-C NUM 匹配上下文分别显示NUM行
					-i 不区分大小写搜索
					-h 查询多个文件时不显示文件名
					-l 查询多个文件时只输出包含匹配字符的文件名
					-L 查询多个文件时只输出包含不匹配字符的文件名
					-n 显示匹配的行及行号
					-b 在输出每一行前显示搜索字符串的行在文件中的偏移量
					-s 不显示不存在或无匹配文件的错误信息
					-w 只匹配一个单词
					-x 只显示整行严格匹配的行
					-v 显示不包含匹配
			b 规则表达式方式的复杂搜索
				正则表达式的主要参数：
					\  忽略正则表达式中特殊字符原有的含义
					^  正则表达式的行开始内容
					$  正则表达式的行结束内容
					\< 正则表达式的某个单词的前面部分开始的行
					\> 正则表达式的某个单词的后面部分开始的行
					[] 正则表达式的某单个字符， a-z A-Z 0-9
					[-]表示匹配范围
					\b 表示匹配的分解符
					a\{m\} 表示匹配重复字符a, m次的行
					a\{m,\} 表示匹配重复字符a, 大于等于m次的行
					a\{m,n\}表示匹配重复字符a, 大于等于m次且小于等于n次的行
					* 匹配0个或者多个字符
					. 匹配任意单个字符
	2 sed(stream editor)

十一 文件链接：ln(link)
	硬链接 软连接
	为文件建立在其他路径中的访问方法（链接）
	语法：ln [option] <源文件> <新建链接文件>
	1 硬链接
	  ln <源文件> <新建文件链接>
	  属性：
		1)创建的硬链接与源文件有相同的inode及date block
		2)创建硬链接时，只能对已存在的文件进行创建
		3)不能交叉文件系统进行硬连接的创建
		4)不能对目录进行创建硬链接，只能对文件创建
		5)删除一个硬链接文件，并不影响其他有相同inode号的文件
	2 软链接
		ln [option] <源文件> <新建链接文件>
		选项：-s
		1) 软连接有自己的文件属性及权限
		2) 可以对不存在的文件或目录创建软链接
		3) 软连接可以交叉文件系统
		4) 可以对文件创建也可以对目录创建
		5) 创建了软连接链接数不变
		6) 删除软连接并不影响被指定的文件，若源文件被删，则相关链接将被请出

十三 进程与文件 lsof
	概念：
		1)在Linux系统中，除了进程以外其他一切都可以理解为文件
		2)进程发挥某些功能，必定涉及打开某些文件，这也说明进程和文件存在一定关系
		3)文件：一种存储数据的格式，当文件可执行时，执行起来会产生功能进程
		4)程序：具有可执行权限的代码，程序是静态的
		5)进程：当程序运行起来，为使程序代码发挥作用，会产生一系列的功能进程，进程是动态的
		6)线程：进程产生
		7)一个程序产生一个或多个进程，一个进程会产生一个或多个线程
	lsof(list open file)
		1)用于查看进程打开文件，打开文件的进程，进程打开的端口（TCP/UDP）
		2)可以找回/恢复删除的文件，是一个十分全面的文件监控系统工具
		3) 因为lsof需要访问核心内存和各种文件，所以只有root用户有权限执行
	选项：
		-a [文件] 列出哪些进程打开了后面指定的文件
		-c <进程名> 列出指定的进程打开了那些文件
		-g 列出PGID号进程的详情
		-d <FD> 列出占用指定描述符的进程
		+d <目录> 列出指定目录下被打开的文件
		+D <目录> 递归列出指定目录下被打开的文件
		-i 列出所有网络链接
		-p <进程号> 显示指定进程号打开的文件
		-u 列出哪一个用户使用了哪些进程打开了哪些文件
		-R 列出父进程ID(PID)
		-t 单独列出进程ID一列
		-r N 每隔N秒就不断执行上面指定的lsof命令
	输出：
		1)COMMAND 进程的名称
		2)PID 进程标识符
		3)TID 任务标识符（线程号）
		4)USER进程用户
		5)PGID进程所属组
		6)FD文件描述符，应用程序通过文件描述符识别该进程打开的文件
			a)cwd 表示current work directory，即应用程序打开的当前工作目录
			b)txt 该类型文件式程序代码 例如应用程序二进制文件或共享库
			c)er FD错误信息
			d)ltx 共享库文件
			e)mmap 内存映射设备
			f)pd 父目录
			g)rtd 根目录
			h)0 标准输入
			i)1 标准输出
			j)2 标准错误输出
				u 文件被打开并处于可读取/写入模式
				r 文件被打开并处于只读模式
				w 文件被打开并处于只写模式
				空格 文件状态未知，并没有被锁定
				- 文件状态未知，并被锁定
			k)mem 被打开的文件是从内存中提取的
		7)TYPE
			a)DIR 目录
			b)CHR 字符类型
			c)BLK 设备类型 
			d)UNIX:UNIX域类型
			e)FIFO 先进先出队列
			f)IPV4 网际协议套接字
			g)DEVICE 指定磁盘名称
			h)REG 普通文件
		8)DEVICE 指定磁盘名称
		9)SIZE/OFF 文件大小
		10)NODE 索引节点在磁盘上的标识
		11)NAME 进程打开的文件
